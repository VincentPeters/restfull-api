## Design Context

### Users
University students at KdG (Karel de Grote-Hogeschool, Belgium) taking a REST API course. They use this companion site during lectures (projected on a beamer) and for self-study at home. Their job: understand REST principles, URI conventions, and API tooling well enough to build and consume APIs. Most are early in their developer journey — clarity beats cleverness.

### Brand Personality
**Calm, focused, minimal.** The site should feel like a well-curated notebook — quiet confidence that lets the content breathe. No visual noise, no gamification gimmicks. The editorial "REST Redactie" identity uses warm, natural tones (cream, rust, sage, ochre) rather than typical cold tech-blue palettes, giving it a distinctive, approachable character without being playful.

### Emotional Goal
**Curiosity.** Students should feel drawn to explore — "I want to try the next exercise." Achieve this through clear progression, rewarding feedback on exercises, and discoverable depth rather than flashy animations or badges.

### Aesthetic Direction
- **Tone:** Editorial / magazine — think printed textbook meets modern web
- **Typography:** Fraunces (serif display) for warmth and character, DM Sans (body) for clean readability, JetBrains Mono (code) for technical precision
- **Palette:** Warm cream base (`#F7F3EE`), dark brown text (`#2D2A26`), three semantic accents — rust for actions/links (`#C4573A`), sage for success/correct (`#5B7B6A`), ochre for highlights/warnings (`#D4A843`)
- **Code specimens:** Dark panels (`#2D2A26`) with syntax-highlighted text — the visual anchor of the site
- **Anti-references:** Avoid Duolingo-style gamification, neon tech aesthetics, or cluttered dashboard layouts. This is a focused learning tool, not an app.
- **Theme:** Light mode only (warm cream, not stark white). Beamer mode overrides to high-contrast white for projection.

### Design Principles
1. **Content first** — Every design decision serves readability and comprehension. If it doesn't help students learn, remove it.
2. **Quiet hierarchy** — Use typography weight/size and whitespace to guide the eye, not color or decoration. Reserve color for semantic meaning (accents, feedback states).
3. **Projection-safe** — All designs must remain legible when projected in a bright classroom. Beamer mode is a first-class feature, not an afterthought.
4. **Discoverable depth** — Surface simplicity with layers underneath. Exercises reveal complexity progressively; theory pages reward scrolling with visual variety (diagrams, specimens, tables).
5. **Respectful minimalism** — Minimal doesn't mean empty. Use whitespace intentionally, but give content room to breathe. Every element earns its place.

### Technical Constraints
- Static site (Astro 6, no SSR) — all interactivity is vanilla JS
- Tailwind CSS v4 with CSS custom properties for theming
- No component library — all components are hand-crafted Astro/HTML
- Dutch language throughout (navigation, content, exercises)
- Deployed to GitHub Pages at `/restfull-api/` base path
